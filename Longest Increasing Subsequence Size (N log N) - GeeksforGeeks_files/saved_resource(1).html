                                    void lisNlgN(int[] nums) {
                                    if (nums == null || nums.length == 0)
                                    return;
                                    //list can be naturally an array, but I found using list is easier to understand
                                    ArrayList<integer> auxList = new ArrayList<integer>();

                                    for (int num : nums) {
                                    //adding the num to tail of list if list is empty or num is greater than current tail of list
                                    if (auxList.size() == 0 || num > auxList.get(auxList.size() - 1)) {
                                    auxList.add(num);
                                    } else {
                                    //if number is smaller than current tail of list, try to find a position to insert it to list
                                    int left = 0; //left boundary
                                    int right = auxList.size() - 1;//right boundary. these two make the length of current list
                                    //do binary search to determine position of current num in the list
                                    //you can also use java's binary search method on collections to return expected position
                                    //the position returned indicates the current num is less than the one already there
                                    //overwriting the existing one with this smaller one
                                    //this is also an algorithm of using binary search to determine insertion position of a given number
                                    while (left < right) {
                                    int mid = (left + right) / 2;
                                    if (auxList.get(mid) < num) {
                                    left = mid + 1;
                                    } else {
                                    right = mid;
                                    }
                                    }
                                    //after this, the right is num's expected position in existing list
                                    //overwrite the value in that position: it's smaller than the value being overwritten!
                                    //indices in auxList does not reflects a list of longest increasing sequence, but its size is
                                    //the correct number of LIS
                                    auxList.set(right, num);
                                    }
                                    }

                                    System.out.println(auxList.size());
                                    }

                                                                                Kiwi • 2 months ago

                                                                                I think this is overly complicated with three cases. We can simply create a new array (let's call it "lis") with same length, initialize all values to Integer.MAX_VALUE. Then loop over the given array, for each element in the array, use binary search to find the *first* element in the "lis" array that is greater than the element, and replace it with the element.

                                                                                See below for an example:
                                                                                array: 2, 5, 3, 7, 11, 8, 9
                                                                                lis: m, m, m, m, m, m, m

                                                                                loop starts, i = 0, array[i] = 2
                                                                                lis after loop: 2, m, m, m, m, m, m

                                                                                i = 1, array[i] = 5
                                                                                lis after loop: 2, 5, m, m, m, m, m

                                                                                i = 2, array[i] = 3
                                                                                lis after loop: 2, 3, m, m, m, m, m

                                                                                i = 3, array[i] = 7
                                                                                lis after loop: 2, 3, 7, m, m, m, m

                                                                                i = 4, array[i] = 11
                                                                                lis after loop: 2, 3, 7, 11, m, m, m

                                                                                i = 5, array[i] = 8
                                                                                lis after loop: 2, 3, 7, 8, m, m, m

                                                                                i = 6, array[i] = 9
                                                                                lis after loop: 2, 3, 7, 8, 9, m, m

                                                                                Then we just return the first non-max value index + 1 in the lis array. In the above example, it will be index of 9 which is 4 + 1 = 5. If all values are non-max, then the answer is simply lis.length.

                                                                                The key here is that when we find the first element that is greater than the array element, it could be the new start of the sequence, so we can simply replace the element we found in lis with the new element. If it turns out to be not a new start, it doesn't matter to us since the existing longest sequence doesn't change.

                                                                                I didn't think of this solution myself (it requires some genius mind). I found it on http://www.jiuzhang.com/soluti... and think this solution is simpler to think about and implement than the solution posted here so I want to share with everyone.

                                                                                                Dachuan Huang • 3 months ago

                                                                                                It looks like the algorithm maintains the following loop invariant: the i-th list is always the LIS with length i that has the smallest tail element seen so far. With this invariant, when we meet a new element, all lists are divided into two groups: groups with tail element < new element, groups with tail element > new element. For the first group, except the last list in first group, it's meaningless to add the new element to the tail to any list, because we know they have the smallest tail; for the second group, we cannot add the new element into the tail because the new element is smaller. It is only the list that is at the end of the first group, that after adding new element, might contribute a new LIS. So the algorithm guarantees that every new element can contribute to the searching process of LIS.

                                                                                                                              i have written the following code with the help of vectors and using the three cass as follows:
                                                                                                                              1. If A[i] is smallest among all end candidates of active lists, we will start new active list of length 1.

                                                                                                                              2. If A[i] is largest among all end candidates of active lists, we will clone the largest active list, and extend it by A[i].

                                                                                                                              3. If A[i] is in between, we will find a list with largest end element that is smaller than A[i]. Clone and extend this list by A[i]. We will discard all other lists of same length as that of this modified list.
